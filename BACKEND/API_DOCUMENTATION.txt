┌─────────────────────────────────────────────────────────────────────────────┐
│ AUTH MODULE                                                                 │
└─────────────────────────────────────────────────────────────────────────────┘

► API Name: User Login
  Action: Authenticates user and generates access + refresh tokens
  Endpoint: POST http://localhost/Sam-Backend/BACKEND/public/index.php/api/auth/login
  Access: PUBLIC (Anyone)
  Tables: super_admin_login, users, customer_authentication
  
  Request Body:
  {
    "email": "admin@mail.com",
    "password": "123456",
    "login_type": "SUPER_ADMIN",
    "salon_id": 1
  }
  
  Response:
  {
    "status": "success",
    "data": {
      "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "refresh_token": "abc123...",
      "expires_in": 900
    }
  }

► API Name: Refresh Access Token
  Action: Generates new access token using valid refresh token
  Endpoint: POST http://localhost/Sam-Backend/BACKEND/public/index.php/api/auth/refresh
  Access: PUBLIC (Requires valid refresh_token)
  Tables: refresh_tokens
  
  Request Body:
  {
    "refresh_token": "abc123..."
  }
  
  Response:
  {
    "status": "success",
    "data": {
      "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "refresh_token": "xyz789...",
      "expires_in": 900
    }
  }

► API Name: User Logout
  Action: Revokes refresh token and logs out user
  Endpoint: POST http://localhost/Sam-Backend/BACKEND/public/index.php/api/auth/logout
  Access: PUBLIC (Requires valid refresh_token)
  Tables: refresh_tokens
  
  Request Body:
  {
    "refresh_token": "abc123..."
  }
  
  Response:
  {
    "status": "success",
    "message": "Logged out successfully"
  }

► API Name: Get Current User
  Action: Returns authenticated user details from JWT token
  Endpoint: GET http://localhost/Sam-Backend/BACKEND/public/index.php/api/auth/me
  Access: AUTHENTICATED (All roles)
  Tables: N/A (Reads from JWT token)
  Headers: Authorization: Bearer <access_token>
  
  Response:
  {
    "status": "success",
    "data": {
      "user_id": 1,
      "role": "ADMIN",
      "salon_id": 1,
      "customer_id": null
    }
  }

► API Name: Generate Subscription Invoice
  Action: Generates invoice for salon subscription billing (frontend calculates, backend stores)
  Endpoint: POST http://localhost/Sam-Backend/BACKEND/public/index.php/api/super-admin/subscriptions/{subscription_id}/generate-invoice
  Access: SUPER_ADMIN
  Tables: invoice_salon, salon_subscriptions, subscription_plans

  Description:
  - Frontend calculates billing amount based on plan type (flat, per-appointment, percentage)
  - Backend receives pre-calculated amounts and stores invoice
  - Usage data stored in notes column as JSON
  - Returns 409 if invoice already exists for same subscription + billing month

  Request Body:
  {
    "billing_month": "2025-02",
    "invoice_date": "2025-03-01",
    "due_date": "2025-03-08",
    "amount": 2254.24,
    "tax_amount": 405.76,
    "total_amount": 2660.00,
    "total_appointments": 45,
    "total_revenue": 45080.00,
    "calculation_breakdown": {
      "base_amount": 0,
      "per_appointment_amount": 2254.24,
      "percentage_amount": 0,
      "subtotal_amount": 2254.24,
      "tax_rate": 18
    }
  }

  Response (Created - 201):
  {
    "status": "success",
    "data": {
      "invoice_salon_id": 25,
      "invoice_number": "INV-SUB-1-20250301-0001",
      "subscription_id": 1,
      "salon_id": 1,
      "billing_month": "2025-02",
      "amount": 2254.24,
      "tax_amount": 405.76,
      "total_amount": 2660.00,
      "payment_status": "UNPAID",
      "due_date": "2025-03-08"
    }
  }

  Response (Already Exists - 409):
  {
    "status": "error",
    "message": "Invoice already exists for this subscription and billing month",
    "data": {
      "invoice_salon_id": 25
    }
  }

  Validation Rules:
  - billing_month: Required, format YYYY-MM
  - total_amount: Required, must be > 0
  - amount: Optional, defaults to 0 (for percentage-only plans)
  - tax_amount: Optional, defaults to 0
  - invoice_date: Optional, defaults to today (YYYY-MM-DD)
  - due_date: Optional, defaults to +7 days (YYYY-MM-DD)
  - total_appointments: Optional, stored in notes for reference
  - total_revenue: Optional, stored in notes for reference
  - calculation_breakdown: Optional, stored in notes for reference

► ENUM VALUES REFERENCE
─────────────────────────────────────────────────────────────────────────────

incentive_type:
  - SERVICE_COMMISSION  → Commission for performing a service
  - BONUS               → Special reward (not service-based)
  - TARGET_ACHIEVEMENT  → Reward for achieving goals/targets

calculation_type:
  - FIXED               → Fixed amount (e.g., ₹50 per service)
  - PERCENTAGE          → Percentage of base amount (e.g., 10% of service price)

status (incentives):
  - PENDING             → Earned but not yet paid
  - PAID                → Earned and paid

payment_mode (incentive_payouts):
  - CASH                → Cash payment
  - UPI                 → UPI transfer (GPay, PhonePe, Paytm)
  - BANK                → Bank transfer (NEFT/IMPS)
  - CHEQUE              → Cheque payment

► SCENARIO 1: SERVICE COMMISSION (FIXED) - Per Service Payment
─────────────────────────────────────────────────────────────────────────────
Use Case: Staff gets fixed ₹50 for every haircut performed

API Used: POST /api/staff/incentives

Request Body:
{
  "staff_id": 1,
  "appointment_id": 10,           // ← Specific appointment (REQUIRED)
  "incentive_type": "SERVICE_COMMISSION",
  "calculation_type": "FIXED",
  "fixed_amount": 50.00,
  "incentive_amount": 50.00,
  "remarks": "Haircut commission - ₹50 per service",
  "status": "PENDING"
}

Database Record:
┌────────────────────────────────────────────────────────────────────────────┐
│ incentives Table                                                           │
├──────────┬──────────┬──────────────┬──────────────┬──────────┬────────────┤
│ incentive│ staff_id │ appointment_ │ incentive_   │ amount   │ status     │
│ _id      │          │ id           │ type         │          │            │
├──────────┼──────────┼──────────────┼──────────────┼──────────┼────────────┤
│ 1        │ 1        │ 10           │ SERVICE_     │ 50.00    │ PENDING    │
│          │          │              │ COMMISSION   │          │            │
└──────────┴──────────┴──────────────┴──────────────┴──────────┴────────────┘

When to Use:
- Fixed commission per service (₹50 per haircut)
- Same amount regardless of service price
- Simple calculation, easy to track

Payment Flow:
1. Service completed → Create incentive (PENDING)
2. End of month → Create payout → Status = PAID

─────────────────────────────────────────────────────────────────────────────

Request Body:

│          │          │              │ COMMISSION   │          │            │
└──────────┴──────────┴──────────────┴──────────────┴──────────┴────────────┘

Calculation Formula:
  incentive_amount = base_amount × (percentage_rate / 100)
  500.00 = 5000.00 × (10.00 / 100)

Important Notes:
- percentage_rate stored as NUMBER (10.00, not "10%")
- Frontend displays with % sign: {percentage_rate}%
- base_amount is typically the service price
- Used for high-value services where commission varies

When to Use:
- Percentage-based commission (5%, 10%, 15% of service price)
- High-value services (bridal, spa packages)
- Incentive scales with service price

► PAYMENT FLOW - FROM EARNING TO PAYMENT
─────────────────────────────────────────────────────────────────────────────

Step 1: Create Incentive (Record Earning)
─────────────────────────────────────────────────────────────────────────────
API: POST /api/staff/incentives

{
  "staff_id": 1,
  "appointment_id": 10,
  "incentive_type": "SERVICE_COMMISSION",
  "calculation_type": "FIXED",
  "incentive_amount": 50.00,
  "remarks": "Haircut commission",
  "status": "PENDING"
}

Result:
- Incentive recorded in database
- status = 'PENDING' (earned but not paid)
- Shows in outstanding incentives report


Step 2: Create Payout (Record Payment)
─────────────────────────────────────────────────────────────────────────────
API: POST /api/staff/incentives/{incentive_id}/payout

Request Body (staff_id NOT required - auto-filled from incentive):
{
  "payout_amount": 50.00,
  "payout_date": "2025-02-28",
  "payment_mode": "BANK",
  "transaction_reference": "TXN123456",
  "remarks": "Paid via bank transfer"
}

Result:
- Payout record created in incentive_payouts table
- Incentive status updated to 'PAID'
- Money transferred to staff


IMPORTANT: One Payout Per Incentive
─────────────────────────────────────────────────────────────────────────────
Current System Design:
- 1 incentive = 1 payout (1:1 relationship)
- Cannot split single incentive into multiple payouts
- Cannot combine multiple incentives into single payout

Monthly Payment Scenario:
If staff has 5 pending incentives at month-end:
- Admin must create 5 separate payout records
- Each payout has its own transaction reference
- All can use same payout_date and payment_mode

Example (Priya's February 2025):
  Incentive #1 (₹50)  → Payout #1 → TXN001 → PAID
  Incentive #2 (₹150) → Payout #2 → TXN002 → PAID
  Incentive #3 (₹500) → Payout #3 → TXN003 → PAID
  Incentive #4 (₹1000)→ Payout #4 → TXN004 → PAID
  Incentive #5 (₹2000)→ Payout #5 → TXN005 → PAID
  
  TOTAL: ₹3,700 paid via 5 separate transactions

► OUTSTANDING CALCULATION
─────────────────────────────────────────────────────────────────────────────

Formula:
  Outstanding = Total Earned - Total Paid

SQL Query:
  SELECT 
    SUM(i.incentive_amount) AS total_earned,
    COALESCE(SUM(ip.payout_amount), 0) AS total_paid,
    SUM(i.incentive_amount) - COALESCE(SUM(ip.payout_amount), 0) AS outstanding
  FROM incentives i
  LEFT JOIN incentive_payouts ip ON i.incentive_id = ip.incentive_id
  WHERE i.staff_id = ?;

Status-Based Query:
  -- Pending incentives (not yet paid)
  SELECT SUM(incentive_amount) 
  FROM incentives 
  WHERE staff_id = ? AND status = 'PENDING';
  
  -- Paid incentives
  SELECT SUM(incentive_amount) 
  FROM incentives 
  WHERE staff_id = ? AND status = 'PAID';
